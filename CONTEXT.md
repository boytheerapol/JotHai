# CONTEXT — JotHai (จดให้) · LINE Expense Tracker (glossary)

> **JotHai** = "จดให้" — เล่นคำกับ *jot* (จดสั้นๆ เร็วๆ) ในภาษาอังกฤษ

> ภาษากลางของโปรเจกต์ ไฟล์นี้เป็น **glossary เท่านั้น** ไม่ใส่ implementation/โค้ด
> decision เชิงสถาปัตยกรรมอยู่ใน `docs/adr/`

---

## Core entities

### Entry (รายการ)
หน่วยข้อมูลพื้นฐานหนึ่งบรรทัด = การเงินหนึ่งครั้งที่ถูกบันทึก ประกอบด้วย: เจ้าของ (User),
ประเภท (Type), จำนวนเงิน, คำอธิบาย, หมวด (Category), แท็ก (Hashtag), เวลา, สถานะ (Status) และ Entry ID
หนึ่ง Entry ผูกกับ **หนึ่ง User เสมอ** และมี **หนึ่ง Category** เท่านั้น
ข้อความหนึ่งที่ส่งมา = หนึ่ง Entry เสมอ (ไม่แยกหลายรายการจากข้อความเดียวใน MVP)

### Status (สถานะของ Entry)
ค่าได้ **active** หรือ **deleted** การลบเป็น **soft delete** — ตั้ง status เป็น `deleted`
ไม่ลบ row จริง เพื่อให้ **undo** ได้และเก็บ audit ทุก Overview/list นับเฉพาะ `active`

### Entry ID
รหัสไม่ซ้ำของแต่ละ Entry ใช้อ้างอิงเพื่อ **แก้/ลบย้อนหลัง** ผ่าน Receipt
มีอยู่ได้เพราะระบบเลือก Optimistic save (บันทึกก่อน แก้ทีหลัง) — ดู ADR-0003

### Type (ประเภท)
ค่าได้เพียง **Income (รายรับ)** หรือ **Expense (รายจ่าย)** เท่านั้น
ถ้า AI ไม่มั่นใจจะ default เป็น Expense

### Category (หมวด)
การจัดกลุ่ม Entry แบบ **หนึ่งต่อหนึ่ง** (1 Entry ได้ 1 Category) เลือกจาก **ชุดที่กำหนดไว้** (fixed)
แต่ชุดนี้ **ขยายได้** โดยแก้ config tab `Categories` ใน Sheet โดยไม่แตะโค้ด
AI (Gemini) มีหน้าที่ map ข้อความเข้ากับ Category ที่มีอยู่ ถ้าไม่เข้าหมวดไหนเลย → `อื่นๆ`
หมายเหตุ: Category เป็นคนละมิติกับ Hashtag
ค่า Category ใน Entry เป็น **snapshot** (เก็บชื่อ ณ ตอนบันทึก) — แก้/ลบ Category ใน config
ไม่ย้อนไปเปลี่ยน Entry เก่า ประวัติจึงไม่เพี้ยนและไม่ต้อง migration

### Hashtag (แท็ก)
label เสริมที่ **User พิมเอง** ด้วยเครื่องหมาย `#` ได้ **หลายอันต่อ Entry** (0..n)
ใช้ group ข้ามหมวด เช่น `#ทริปเชียงใหม่` `#ของขวัญแม่`
ต่างจาก Category: Category เป็น auto/หนึ่งอัน/มาจาก fixed list — Hashtag เป็น manual/หลายอัน/อิสระ
Hashtag เป็น **มิติกรองได้** ใน LIFF (เลือก tag แล้วดูสรุปเฉพาะ tag นั้น) ไม่ใช่แค่ป้ายนิ่งๆ
เก็บแบบ **normalize**: ตัด `#`, trim, ตัวอักษรอังกฤษเป็น lowercase (ไทยคงเดิม) เพื่อกันกรองแตก

### Clarification (การถามกลับ)
สถานะชั่วคราวเมื่อข้อความที่ User ส่งมา **ไม่มีจำนวนเงินชัดเจน** บอทจะ **ไม่บันทึก**
แต่ตอบกลับถามจำนวน แล้วจำไว้ว่ากำลังรอคำตอบของ Entry ไหน เมื่อ User ตอบจำนวนมา
จึงประกอบเป็น Entry ให้ครบ (state ผูกกับ userId, อายุสั้น)

### Receipt (ใบยืนยัน)
การ์ดที่บอทส่งกลับ **หลังบันทึก Entry ลง Sheet แล้ว** แสดงสิ่งที่ AI parse ได้
(Type, จำนวน, คำอธิบาย, Category, Hashtag) พร้อมปุ่มแก้/ลบ ที่อ้างถึง Entry ID
*ไม่ใช่* ขั้นตอนยืนยันก่อนบันทึก (ระบบไม่มีขั้นตอนนั้น — ดู ADR-0003)

### Overview (ภาพรวม)
มุมมองสรุปที่ User เปิดดูเองผ่านเมนู (LIFF) มี 3 chart (donut) + 1 list:
1. ภาพรวมรายรับ–รายจ่าย ของเดือน
2. รายรับ แยกตาม Category
3. รายจ่าย แยกตาม Category
4. **รายการ (Entry list)** — list รายตัวของเดือน กด **แก้ทุก field / ลบ (soft) / undo** ได้ในนั้น
ขอบเขตเวลาเริ่มต้น = เดือนปฏิทิน (Asia/Bangkok) แสดงเฉพาะ Entry `active` ของ User คนนั้น

### Receipt list (tab รายการใน LIFF)
ช่องทาง**หลัก**ในการแก้รายการย้อนหลังหลังจาก Receipt ในแชตเลื่อนหายไป
ต่างจาก Receipt (การ์ดในแชตที่เด้งทันทีหลังบันทึก) — list นี้เปิดดูเมื่อไหร่ก็ได้จากเมนู

### Help / Welcome
ข้อความสอนวิธีใช้ บอทส่ง **Welcome** อัตโนมัติเมื่อ Admin เปลี่ยนสถานะเป็น `approved`
และผู้ใช้พิม **"help"** เพื่อเรียกดูวิธีใช้ซ้ำได้ทุกเมื่อ

## Users & access

### User (ผู้ใช้ / บัญชี)
หนึ่งคน = หนึ่ง LINE account ระบุตัวด้วย **LINE userId** (string ลับ เช่น `U4af49...`)
ระบบเป็น **multi-user แยกบัญชี**: ข้อมูลและ Overview ของแต่ละ User แยกขาดจากกัน

### Whitelist / สถานะผู้ใช้
User มีสถานะ `pending` หรือ `approved` ใน tab `Users`
- userId ได้ **ก็ต่อเมื่อ** คนนั้นทักบอทมาแล้ว (มากับ webhook) → ใส่ล่วงหน้าไม่ได้
- คนใหม่ทักมา → ระบบ log เป็น `pending` อัตโนมัติ + ตอบว่ายังไม่ได้รับอนุญาต
- **Admin** เปลี่ยนสถานะเป็น `approved` เองใน Sheet จึงใช้งานได้ (ดู ADR ไม่จำเป็น — เป็น config)

### Admin
ผู้ดูแล (เจ้าของระบบ) อนุมัติ User ด้วยการแก้ค่าสถานะใน Sheet โดยตรง ไม่มีคำสั่ง admin ใน LINE (MVP)

## Channels & runtime (สรุปสั้น — รายละเอียดใน ADR)

### Backend
รันบน **Google Apps Script (GAS)** — webhook ผ่าน `doPost`, bind กับ Google Sheet โดยตรง (ADR-0001)

### LIFF
หน้าเว็บที่เปิด **ภายใน LINE app** ใช้ render Overview/Chart แบบ interactive (ADR-0002)

### Reply vs Push
ระบบส่งข้อความแบบ **Reply เท่านั้น** (ตอบกลับ event) เพื่ออยู่ใน free tier — ไม่มี push สรุปอัตโนมัติ (ADR-0004)
